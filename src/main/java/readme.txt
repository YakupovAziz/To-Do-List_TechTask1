Тестовое задание Java Junior

Цель: Разработать простой RESTful API для управления задачами To-Do List

Задачи:

1. Реализовать запрос для получения выходных дней с помощью внешнего API.
Примеры API: https://date.nager.at/Api. будет плюсом: Полученные выходные дни необходимо записывать в кэш.

2. Создание задачи
   Реализовать endpoint для создания новой задачи.
   Задача должна содержать следующие поля:
ID (генерируется автоматически)
Заголовок
Описание
Дата создания
Статус(по умолчанию “не выполнено”)
Предполагаемая дата завершения задачи

Предусмотреть валидацию: дата завершения задачи не может попадать на выходной или праздничный день. В таком случае возвращать клиенту ошибку “выберите другой день — ближайший доступный {date}”, date - необходимо высчитать и подставить в сообщение. Предусмотреть сохранение в бд.

3. Получение списка задач
	Реализовать endpoint для получения списка всех задач. Возможность фильтрации по статусу задачи (“выполнено”, “не выполнено”). Опционально: реализовать пагинацию

4. Обновление задачи
	Реализовать endpoint для обновления существующей задачи. Можно обновить заголовок, описание и статус задачи.

5. Удаление задачи
	Реализовать endpoint для удаления задачи по её ID.

	Технологический стек: Java 21+, Spring Boot 3+, Spring Data JPA. Можно использовать in-memory базу данных, например, H2.
	Использование Mysql/Postgres будет плюсом. Maven/Gradle. JUnit 5 для тестирования. Webclient/feign/restTemplate

	Дополнительно:
1. 	Написать 1 unit тест
2. 	Реализовать обработку ошибок (например, задача с данным ID не найдена).
3. 	Задокументировать API с использованием Swagger/OpenAPI.
